# Code of Conduct

## Our Pledge

We as contributors and maintainers pledge to make participation in bvlos-sim a
respectful, harassment-free experience for everyone.

This project is focused on deterministic BVLOS mission validation. Technical
disagreement is expected and welcome, but it must remain constructive,
evidence-based, and respectful.

## Expected Behavior

Examples of behavior that supports a healthy project community include:

- using welcoming and inclusive language
- respecting different experience levels and viewpoints
- giving and receiving technical feedback professionally
- focusing criticism on code, documentation, designs, and tradeoffs rather than
  people
- accepting responsibility for mistakes and correcting them
- prioritizing the safety and well-being of the community

## Unacceptable Behavior

Examples of unacceptable behavior include:

- harassment, intimidation, stalking, or sustained disruption
- insulting, demeaning, or discriminatory comments
- sexualized language or imagery
- publishing private information without explicit permission
- personal attacks, trolling, or inflammatory comments
- abuse of project processes, issue trackers, reviews, or discussions
- any conduct that would reasonably be considered inappropriate in a
  professional open-source setting

## Scope

This Code of Conduct applies in all project spaces, including the repository,
issues, pull requests, discussions, documentation, release channels, and any
community spaces that represent the project.

It also applies when an individual is representing the project publicly, such as
using an official project account, presenting project work, or acting as a
maintainer.

## Enforcement

Maintainers are responsible for clarifying and enforcing acceptable behavior.
They may remove, edit, or reject comments, commits, issues, pull requests, and
other contributions that do not align with this Code of Conduct.

Depending on the severity and pattern of behavior, maintainers may take actions
including:

- a private or public clarification
- a warning
- temporary restriction from project spaces
- permanent removal from project spaces

Enforcement decisions should be proportionate, consistent, and focused on
protecting the project community.

## Reporting

If you experience or witness unacceptable behavior, report it through the
project's GitHub issue or discussion channels. If the report is sensitive, avoid
posting private details publicly and contact a maintainer through an appropriate
private GitHub channel when available.

Reports should include:

- what happened
- where and when it happened
- who was involved
- any relevant links, screenshots, or context

Maintainers will review reports as promptly as practical and handle sensitive
information with care.

## Attribution

This Code of Conduct is adapted from common open-source community standards,
including the Contributor Covenant, with project-specific wording for bvlos-sim.
